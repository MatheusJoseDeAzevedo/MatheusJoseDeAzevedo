1) Introdução

Em programação, nem tudo acontece em linha reta. Muitas vezes precisamos decidir o que fazer e também repetir ações sem ficar copiando código.

Objetivo deste material

Explicar, de forma didática, duas ideias fundamentais: condicionais (decisão) e laços (repetição).

No final, você encontra dois formulários: um para avaliar o conteúdo e outro para avaliar a página.

Dica rápida

Condicionais escolhem caminhos. Laços repetem passos.

2) Condicionais (if / else)

O que é?

Uma condicional serve para tomar uma decisão baseada em uma condição que pode ser verdadeira ou falsa. 

// Pseudocódigo
se (idade >= 18) então
  escreva("Maior de idade")
senão
  escreva("Menor de idade")
fimse

Exemplo bem comum: se a média for maior ou igual a 6, o aluno é aprovado; senão, é reprovado.

Exemplo

Se média ≥ 6 então Aprovado
Senão Reprovado 

leia(media)
se (media >= 6) então
  escreva("Aprovado")
senão
  escreva("Reprovado")
fimse

Observação: o importante é entender a lógica. A linguagem pode mudar, mas a ideia é a mesma.

3) Laços (for / while)

Para que servem?

Laços são usados quando precisamos repetir um conjunto de instruções. Por exemplo: somar números, ler várias entradas, percorrer um vetor, validar uma senha.

Quando usar cada um?

  • for: quando existe um contador e a quantidade de repetições é controlada.
  • while: quando a repetição depende de uma condição (pode variar).

Exemplos

for: repetir do 1 ao 5 (contador aumentando)
while: repetir enquanto a senha estiver errada 

// for: repetição com contador
para i de 1 até 5 faça
  escreva(i)
fimpara

// while: repetição por condição
enquanto (senha != correta) faça
  leia(senha)
fimenquanto

Erro comum

Em laços, se a condição nunca mudar, pode acontecer um loop infinito laço infinito. Por isso é importante atualizar o contador ou a condição.

4) Tabela comparativa

Resumo simples para lembrar:

Estrutura Função Quando usar
if / else Decisão Quando existe condição (sim/não)
for Repetição com contador Quando a quantidade de repetições é controlada
while Repetição por condição Quando a repetição depende de uma condição variável

Mini revisão

  1. Condicionais: escolha um caminho.
  2. Laços: repita uma ação.
  3. Organize sua lógica antes de codar.
